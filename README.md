# Система MDM (master data management). Подсистема RDM (Reference Data Management)
Небольшой учебный проект по ведению справочников.

## Цель проекта: 
Система MDM (master data management). Подсистема RDM (Reference Data Management)

## Задачи проекта:
1. Сделать систему регистрации и авторизации пользователей
2. Реализовать интерфейс пользовательского ввода для описания справочников.
3. Реализовать функции позволяющие создавать таблицу в БД, а также регистрировать модель справочника, на основе описания пользователя. 
4. Реализовать функции позволяющие наполнять созданный справочник информацией
5. Изучить возможности фреймворка Django и применить их на практике

## Требуемый функционал
1. Управление пользователями. Вход / Выход / Регистрация
2. Главная страница
   - Отображение зарегистрированных справочников
   - Меню
   - Список незарегистрированных справочников
   - Список динамически зарегистрированных таблиц 
   - Административная панель
   - Поисковая строка по зарегистрированным справочникам
3. Проверка принадлежности к роли, в зависимости от этого отображается разный интерфейс и только те справочники которые относятся к конкретному пользователю
4. Функционал работы со справочниками
   - Страница заведения колонок объекта
   - Функция согласования справочника
   - Функция регистрации справочника (только для Администратора)
   - Функция отзыва справочника из согласования
   - Функция удаления справочника из регистрации
   - Сервис / модуль по работе со справочниками
     * Добавление записи
     * Удаление записи
     * Редактирование записи
     * Отображение данных справочника
     * Отображение структуры зарегистрированной таблицы

## Основной фреймворк для реализации проекта
1. Фреймворк Django - выбран потому что он позволяет достаточно быстро реализовать приложение, содержит очень много встроенных функций.  
А также одной из поставленной задачей было изучение этого инструмента.

## Улучшения, развитие проекта
Есть очень много вариантов развития проекта, основные направления по backend приложения:
1. Расширенная панель пользователя / администратора
2. Передача прав между ролями
3. Создание и управление древовидным отображением папок и справочников (панель привязки / отвязки справочников)
4. Проверки на формах (валидация типов данных, конфликты типов данных)
5. Введение дополнительных атрибутов (отображение полей справочника, внешние ключи, ведение дополнительных кнопок с действиями)
6. Древовидные справочники
7. Ведение справочника в EAV модели
8. Управление справочником через функционал БД (приложение как прослойка и отображение)
9. Локализация

## Составляющие:

1. `core` - основной модуль с настроками приложения
2. `meta_app` - приложениие для описания основных метаданных справочников
2. `user_app` - приложениие для управления пользователями (регистарция, авторизация)
3. `dynamic_tables_app` - приложениие для управления созданием и регистрации динамических справочников
4. `db_init` - специальная папка со скриптами при первом запуске БД. Служит для инициализации
5. `docker-compose.yaml` - Docker compose для поднятия БД PostgreSQL

## Порядок установки проекта

1. Клонировать код проекта в нужную папку
2. Необходимо создать виртуальное окружение в проекте: ```python3 -m venv .venv```
3. Активировать виртуальное окружение: ```source .venv/bin/activate```
4. Установить менеджер пакетов UV: ```pip install uv```
5. Выполнить синхронизацию установленных библиотек: ```uv sync```
6. Поднять БД, путем запуска docker контейнера: ```docker compose --env-file ./core/.env ud -d```
7. Выполнить начальную миграцию Django: ```python3 manage.py initialization```
8. Выполнить начальную миграцию данных в БД: ```python3 manage.py meta_app_init```
9. Создать суперпользователя: ```python3 manage.py createsuperuser```
10. Запустить проект: ```python3 manage.py runserver```
11. Зайти под администратором и создать две группы: 
    - Архитектор
    - Пользователь
    
**Примечание**: Соответствено в группу архитектора добавить текущего пользователя + раздать права
